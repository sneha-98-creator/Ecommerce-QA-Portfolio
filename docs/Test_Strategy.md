# Test Strategy

## Objective

The objective of this strategy is to define the testing methodology for ShopEasy.

---

# Testing Levels

- Unit Testing (Development Team)
- Integration Testing
- System Testing
- User Acceptance Testing

---

# Functional Testing

Verify every feature behaves according to business requirements.

Modules covered:

- Login
- Registration
- Search
- Cart
- Checkout
- Payment

---

# Smoke Testing

Performed after every new build deployment.

Smoke Suite includes:

- Login
- Search
- Add to Cart
- Checkout
- Logout

---

# Sanity Testing

Performed after bug fixes.

Areas covered:

- Fixed defects
- Related functionality

---

# Regression Testing

Complete regression executed before every release.

Includes:

- Login
- Registration
- Search
- Cart
- Payment
- Orders
- Profile

---

# API Testing

Validate:

- Response Code
- Response Time
- Request Validation
- Authentication
- Error Handling

---

# Defect Lifecycle

New

↓

Assigned

↓

In Progress

↓

Fixed

↓

Retest

↓

Closed

---

# Test Deliverables

- Test Plan
- Test Cases
- Test Scenarios
- RTM
- Bug Reports
- Test Summary Report

---

# Test Metrics

- Test Case Execution %
- Defect Density
- Defect Leakage
- Pass Percentage
- Requirement Coverage
