# Functional Requirement Specification (FRS)

## Registration Module

| ID | Requirement |
|----|-------------|
| FR-001 | User should register with a unique email address. |
| FR-002 | Mandatory fields must be validated. |
| FR-003 | Password should meet security requirements. |
| FR-004 | Registration confirmation email should be sent. |

---

## Login Module

| ID | Requirement |
|----|-------------|
| FR-005 | User should log in with valid credentials. |
| FR-006 | Password should remain masked. |
| FR-007 | Error message should appear for invalid login. |
| FR-008 | Forgot Password should allow password reset. |

---

## Search Module

| ID | Requirement |
|----|-------------|
| FR-009 | Search by product name. |
| FR-010 | Filter by category. |
| FR-011 | Sort by price. |
| FR-012 | Sort by ratings. |

---

## Shopping Cart

| ID | Requirement |
|----|-------------|
| FR-013 | Add products to cart. |
| FR-014 | Update product quantity. |
| FR-015 | Remove products from cart. |
| FR-016 | Display cart total. |

---

## Checkout

| ID | Requirement |
|----|-------------|
| FR-017 | Select shipping address. |
| FR-018 | Select payment method. |
| FR-019 | Review order summary. |
| FR-020 | Confirm order successfully. |

---

## Payment

| ID | Requirement |
|----|-------------|
| FR-021 | Accept credit/debit cards. |
| FR-022 | Accept UPI payments. |
| FR-023 | Display successful payment confirmation. |
| FR-024 | Generate Order ID after successful payment. |
